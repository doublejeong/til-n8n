### Context
HTML 클래스명은 단순히 CSS 스타일을 적용하는 것을 넘어, 웹 페이지의 구조와 의미를 명확하게 드러내는 핵심적인 역할을 수행합니다. 일관성 없는 클래스 명명은 코드의 가독성을 저해하고, 유지보수를 어렵게 하며, 특히 대규모 프로젝트나 협업 환경에서 예측 불가능한 스타일 충돌을 야기할 수 있습니다. 따라서 효율적이고 체계적인 클래스 명명 규칙을 적용하는 것은 고품질의 웹 개발을 위한 필수적인 요소입니다.

### Core
HTML 클래스명을 작성할 때 고려해야 할 주요 규칙과 대표적인 방법론은 다음과 같습니다.

1.  **일반적인 권장 사항**
    *   **의미 기반(Semantic):** 클래스명은 요소의 시각적인 특징(예: `red-text`, `float-left`)보다는 역할과 목적을 나타내도록 작성해야 합니다. 이는 HTML과 CSS의 관심사를 명확히 분리하고, 스타일 변경 시에도 클래스명을 유지할 수 있도록 돕습니다.
        *   **좋은 예:** `user-profile`, `main-navigation`, `button-primary`
        *   **나쁜 예:** `text-red`, `left-column`, `big-button`
    *   **소문자 및 하이픈(Kebab-case):** 여러 단어로 이루어진 클래스명은 소문자와 하이픈(-)을 사용하여 연결하는 것이 일반적인 관례입니다. 이는 가독성을 높이고 일관된 스타일을 유지하는 데 매우 효과적입니다.
        *   **좋은 예:** `product-item`, `cart-button`
        *   **나쁜 예:** `productItem`, `Product_Item`
    *   **간결하고 명확하게:** 너무 길거나 모호한 이름, 또는 의미를 파악하기 어려운 약어 사용은 피해야 합니다. 클래스의 역할을 명확하게 설명하면서도 간결하게 작성하는 것이 중요합니다.
    *   **재사용성 고려:** 특정 요소에만 국한되지 않고, 여러 곳에서 재사용할 수 있는 일반적인 이름을 지향하여 컴포넌트 기반 개발 및 모듈화된 스타일시트 작성을 용이하게 합니다.
    *   **상태 표현:** `is-active`, `has-error`, `is-loading`, `is-collapsed` 등 상태를 나타내는 접두사를 활용하여 동적인 상태를 명확히 표현할 수 있습니다. 이는 JavaScript를 통해 동적으로 클래스를 제어할 때 유용합니다.
    *   **숫자로 시작하지 않기:** 클래스명은 숫자로 시작할 수 없습니다. 이는 CSS 식별자 규칙에 위배되며, 유효성 검사에 실패할 수 있습니다.

2.  **대표적인 방법론: BEM (Block-Element-Modifier)**
    BEM은 대규모 프로젝트에서 CSS를 구조화하고 재사용성을 극대화하기 위해 널리 사용되는 명명 규칙입니다. `block__element--modifier` 형식을 따르며, CSS 스코프 충돌을 방지하고 코드의 가독성을 높이는 데 기여합니다.

    *   **Block (블록):** 독립적으로 재사용 가능한 컴포넌트나 섹션의 이름입니다. CSS 선택자로 사용할 때는 점(.)으로 시작합니다. 블록은 주변 환경의 영향을 받지 않아야 하며, 어디서든 재사용 가능하도록 설계됩니다.
        *   예: `.header`, `.menu`, `.button`, `.card`
    *   **Element (요소):** 블록의 일부로, 블록 없이는 독립적인 의미가 없는 요소입니다. `__` (두 개의 언더스코어)로 블록과 연결합니다. 요소는 블록 내에서만 의미를 가집니다.
        *   예: `.header__logo`, `.menu__item`, `.button__icon`, `.card__title`
    *   **Modifier (수정자):** 블록이나 요소의 모양, 상태, 행동을 변경하는 추가적인 클래스입니다. `--` (두 개의 하이픈)으로 블록이나 요소와 연결합니다. 수정자는 기존 블록/요소의 클래스와 함께 사용됩니다.
        *   예: `.button--primary` (기본 버튼), `.menu__item--active` (활성화된 메뉴 항목), `.card--disabled` (비활성화된 카드)

    **BEM 예시:**
    ```html
    <div class="card card--featured">
      <img class="card__image" src="product.jpg" alt="제품 이미지">
      <h2 class="card__title">최신 스마트폰</h2>
      <p class="card__description">강력한 성능과 뛰어난 디자인을 자랑합니다.</p>
      <button class="button card__button button--primary button--large">구매하기</button>
    </div>
    ```

    **BEM의 장점:**
    *   **명확한 구조:** 클래스명만으로 HTML 구조와 컴포넌트의 관계를 쉽게 파악할 수 있습니다.
    *   **재사용성:** 각 블록과 요소가 독립적으로 설계되어 재사용이 용이합니다.
    *   **충돌 방지:** 고유한 명명 규칙으로 인해 CSS 선택자 간의 충돌 가능성을 크게 줄여줍니다.
    *   **유지보수 용이성:** 특정 스타일 변경 시 어떤 클래스를 수정해야 할지 명확하게 알 수 있습니다.

### Insight
HTML 클래스명 규칙을 일관성 있게 적용하는 것은 단순히 코드를 정리하는 것을 넘어, 프로젝트의 전반적인 품질과 개발 효율성에 지대한 영향을 미칩니다. 특히 BEM과 같은 구조화된 명명 규칙은 대규모 웹 애플리케이션에서 발생할 수 있는 CSS 충돌을 효과적으로 방지하고, 컴포넌트 기반 개발을 용이하게 합니다.

개인 프로젝트나 소규모 팀에서도 이러한 규칙을 적용하면, 장기적으로 코드의 이해도를 높이고 유지보수 비용을 절감할 수 있음을 깨달았습니다. 클래스명을 설계할 때마다 '이 이름이 요소의 역할을 잘 나타내는가?', '재사용 가능한가?', '다른 개발자가 쉽게 이해할 수 있는가?'를 고민하는 습관을 들이는 것이 중요하다고 생각합니다.
