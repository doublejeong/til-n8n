### Context

자바스크립트에서는 `function` 키워드를 사용하여 일반 함수를 정의할 수 있으며, 화살표 함수는 더 간결하고 읽기 쉬운 코드를 제공하기 위해 도입되었습니다. 이 두 종류의 함수는 다양한 면에서 차이가 있어 이해하는 것이 중요합니다.

### Core

화살표 함수와 일반 함수의 주요 차이는 다음과 같습니다:

- **상위 스코프의 `this` 접근**: 화살표 함수에서는 상위 스코프의 `this`를 그대로 상속받습니다. 반면, 일반 함수는 자신이 정의된 위치에서 `this`를 찾아냅니다.

  ```javascript
  const obj = {
    value: 'hello',
    greet: function() {
      console.log(this.value);
    },
    arrowGreet: () => {
      console.log(this.value);
    }
  };

  // 일반 함수의 this: window (브라우저 환경에서)
  obj.greet(); // undefined

  // 화살표 함수의 this: obj
  obj.arrowGreet(); // hello
  ```

- **인수 전달**: 화살표 함수는 `function` 키워드가 아닌 `(param1, param2) => { body }` 형식으로 작성할 수 있습니다. 일반 함수에서는 `function name(param1, param2) {}` 형태로 작성해야 합니다.

  ```javascript
  const log = (message) => {
    console.log(message);
  };

  function printMessage(message) {
    console.log(message);
  }

  // 화살표 함수의 호출: 메시지를 직접 전달합니다.
  log('Hello, world!'); // Hello, world!

  // 일반 함수의 호출: 파라미터로 메시지만 전달해야 합니다.
  printMessage('Hello, world!'); // Hello, world!
  ```

- **문맥 내부에서 작성**: 화살표 함수는 `function` 키워드가 없는 문맥 내에서 작성될 수 있습니다.

  ```javascript
  const log = () => {
    console.log(this);
  };

  // 화살표 함수의 this: 전역 객체 (브라우저 환경)
  log(); // Window {...}

  function logFunction() {
    return () => {
      console.log(this);
    };
  }

  // 일반 함수 내부에서 화살표 함수 작성
  const logInner = logFunction();
  logInner(); // Window {...}
  ```

### Insight

화살표 함수는 코드의 가독성을 높이고, 특정 상황에서는 `this` 스코프 문제를 해결할 수 있어 유용합니다. 하지만 일반 함수는 더 많은 컨텍스트와 유연성을 제공하며, 화살표 함수로 작성하기 어려운 경우가 있습니다.
