### Context
Docker 컨테이너가 이미 실행 중인 상태에서, 컨테이너의 메인 프로세스를 중단하거나 재시작하지 않고 컨테이너 내부로 진입하여 명령어를 실행하고 싶을 때가 있습니다. 예를 들어, 웹 서버 컨테이너가 정상적으로 동작 중인 상태에서 내부 파일 시스템을 확인하거나, 로그를 분석하거나, 특정 스크립트를 실행해야 할 때 유용합니다.

### Core
이미 실행 중인 Docker 컨테이너 내부로 진입하는 가장 보편적이고 정확한 방법은 `docker exec` 명령어를 사용하는 것입니다. 이 명령어는 실행 중인 컨테이너 내부에 새로운 프로세스를 실행시킵니다.

**대화형 셸(Interactive Shell) 진입:**

```bash
docker exec -it [컨테이너_이름_또는_ID] /bin/bash
```

**단일 명령어 실행:**
컨테이너 내부에서 특정 명령어를 한 번만 실행하고 결과를 확인하고 싶을 때도 `docker exec`를 사용할 수 있습니다. 이 경우 `-it` 옵션 없이 명령어를 직접 지정합니다.

```bash
docker exec [컨테이너_이름_또는_ID] [실행할_명령어]
```

**예시: Nginx 컨테이너의 액세스 로그 확인**
```bash
docker exec my-nginx-container cat /var/log/nginx/access.log
```

**명령어 설명:**

*   `docker exec`: 실행 중인 컨테이너 내부에서 명령어를 실행합니다.
*   `-i` (또는 `--interactive`): 표준 입력을 열어둡니다. 셸과 상호작용하기 위해 필요합니다.
*   `-t` (또는 `--tty`): 의사(pseudo)-TTY를 할당합니다. 셸 출력의 서식을 올바르게 표시하기 위해 필요합니다. `-it`를 함께 사용하면 상호작용 가능한 터미널 세션을 얻을 수 있습니다.
*   `[컨테이너_이름_또는_ID]`: 진입하고자 하는 컨테이너의 이름 또는 ID입니다. `docker ps` 명령어로 확인할 수 있습니다.
*   `/bin/bash`: 컨테이너 내부에서 실행할 명령어입니다. 주로 셸(Bash, Sh 등)을 지정하여 상호작용 가능한 환경을 만듭니다. 만약 `/bin/bash`가 컨테이너에 설치되어 있지 않다면 `/bin/sh`를 대신 사용할 수 있습니다.

**사용 단계:**

1.  **실행 중인 컨테이너 확인:**
    먼저 `docker ps` 명령어를 사용하여 현재 실행 중인 컨테이너 목록과 그들의 이름 또는 ID를 확인합니다.

    ```bash
    docker ps
    ```

    출력 예시:
    ```
    CONTAINER ID   IMAGE          COMMAND                  CREATED         STATUS         PORTS     NAMES
    a1b2c3d4e5f6   nginx:latest   "nginx -g 'daemon of…"   2 hours ago     Up 2 hours     80/tcp    my-nginx-container
    ```

2.  **컨테이너 내부로 진입:**
    위 예시에서 `my-nginx-container`라는 이름의 컨테이너로 진입하려면 다음 명령어를 사용합니다.

    ```bash
    docker exec -it my-nginx-container /bin/bash
    ```

    이제 컨테이너 내부의 셸 환경으로 진입했으며, 여기서 다양한 명령어를 실행할 수 있습니다.

3.  **컨테이너에서 나가기:**
    컨테이너 셸에서 나갈 때는 `exit` 명령어를 입력합니다.

    ```bash
    exit
    ```

    이렇게 나가더라도 컨테이너의 메인 프로세스는 계속 실행 중이며, 컨테이너 자체는 중단되지 않습니다.

### Insight
`docker exec`의 핵심은 "실행 중인 컨테이너 내부에 *새로운 프로세스를 추가적으로 실행*한다"는 점입니다. 이는 컨테이너의 주 프로세스(예: 웹 서버, 데이터베이스)와는 별개의 프로세스이므로, 기존 서비스의 중단 없이 안전하게 컨테이너 내부 환경에 접근할 수 있습니다.

만약 컨테이너가 실행 중이 아닌 `Exited` 상태라면 `docker exec` 명령어는 작동하지 않습니다. 이 경우에는 먼저 `docker start [컨테이너_이름_또는_ID]` 명령어로 컨테이너를 시작한 다음, `docker exec`를 사용하여 진입해야 합니다.

간혹 `docker attach` 명령어도 컨테이너 진입을 위해 사용되지만, `attach`는 컨테이너의 메인 프로세스에 연결되므로, 메인 프로세스가 셸이 아니라면 우리가 원하는 상호작용이 불가능하거나, `Ctrl+C` 같은 키 입력이 메인 프로세스에 전달되어 컨테이너가 종료될 위험이 있습니다. 따라서 대부분의 경우 `docker exec`를 사용하는 것이 더 안전하고 권장되는 방법입니다.
