### Context
처음 n8n을 사용하여 워크플로우를 자동화하는 프로젝트를 진행하였습니다. Discord 서버에서 온 메시지를 수신하면 트리거가 발생하고, 이에 따른 agent를 실행하여 TIL 형식의 데이터를 변환한 후 GitHub에 push합니다. push가 완료되면 README.md 파일에 인덱스를 추가하고, 마지막으로 사용자에게 완료 메시지를 발신합니다.

### Core
```plaintext
1. Discord 메시지 수신 트리거 설정
   - Discord 노드를 사용하여 서버의 특정 채널에서 메시지를 수신하도록 합니다.
2. TIL 형식 변환 및 GitHub Push
   - n8n의 "TIL" 노드를 사용하여 입력된 텍스트를 TIL 형식으로 변환합니다.
   - 변환된 내용을 GitHub에 push하기 위한 Git Hub Repository 노드와 Commit Message 노드를 설정합니다.
3. README.md 파일 업데이트
   - push가 완료되면 "README.md" 파일을 읽고, 인덱스를 추가하여 다시 write 합니다.
4. 완료 메시지 발신
   - 모든 과정이 완료되면 Discord 채널에 "작업 완료!"라는 메시지를 발신합니다.
```

### Insight
n8n을 처음 사용해보는 건데, 전체적인 프로세스를 이해하면 그리 어렵지 않게 워크플로우를 만들 수 있습니다. 이 노드들의 input과 output만 정확히 파악하면 쉽게 구성할 수 있었고, 중간중간 모르는 부분은 쳇 GPT의 도움을 받아 응용하여 구현했습니다. 아직 백프로는 아니지만, 지속적으로 학습하면서 완성도를 높여나가려고 합니다.
