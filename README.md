## 🛠 작성 및 관리 규칙

일관된 지식 자산화를 위해 아래 규칙을 기준으로 문서를 작성하고 관리합니다.

### 명명 규칙

- 형식: `YYYY-MM-DD-제목.md`
- 모든 공백은 하이픈(`-`)으로 대체하고, 특수문자는 사용하지 않습니다.
- 예시: `2026-01-30-Clawdbot-프롬프트-최적화-전략.md`

### 디렉토리 구조

- `n8n/`: 워크플로우 설계 및 노드 활용 전략
- `python/`: 파이썬 문법 및 강의용 라이브러리 분석
- `ai-agent/`: LLM 프롬프트 설계 및 에이전트 구축

### 문서 내부 양식

- `Context`: 해결하려는 문제 또는 학습 배경
- `Core`: 실제로 작동하는 코드 스니펫이나 워크플로우 로직
- `Insight`: 실행 결과와 기술적 견해, 그리고 검증 데이터

## 🚀 AI Automation

이 저장소는 Human-in-the-Loop 모델이 적용된 n8n 기반 AI 워크플로우로 운영됩니다.

- n8n 기반 자동화 워크플로우를 직접 설계하고 구축했습니다.
- 미니 PC에 Linux를 설치해 홈서버 환경에서 직접 배포 및 운영했습니다.
- Discord 봇과 연동하여 Discord에서 알림을 받고 상호작용할 수 있도록 구성했습니다.

- 메모 감지: 파편화된 학습 내용을 AI 에이전트가 수집합니다.
- 지능형 정제: 기술적 사실 관계를 검토하고, 표준화된 Markdown 형식으로 초안을 작성합니다.
- 인간 검토: 초안은 직접 검토하고 수정한 뒤, 기록의 진실성과 정확성을 최종 승인합니다.
- 자동 기록: 승인된 문서만 GitHub에 자동 커밋되며, README 인덱스도 함께 업데이트됩니다.
